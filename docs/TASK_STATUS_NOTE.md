# Статусы задач: известное расхождение

## Что происходит

В Swagger-документации (и в `model/task.go`) статусы задачи перечислены как:

```
pending | in_progress | completed | cancelled
```

Но когда задача **только что создана**, API реально возвращает:

```
new
```

Это значение выставляет база данных по умолчанию (`DEFAULT 'new'` в миграции) — приложение его не задаёт явно.

## Почему это не критично

Бэкенд хранит и отдаёт `new` корректно — данные согласованы на всех уровнях. Ошибка только в Swagger-документации (там в примере стоит `pending` вместо `new`).

## Полный список статусов (реальный)

| Статус | Когда |
|--------|-------|
| `new` | Задача только создана (дефолт) |
| `pending` | Выставляется вручную через PATCH /tasks/{id} |
| `in_progress` | Выставляется вручную через PATCH /tasks/{id} |
| `completed` | Выставляется вручную через PATCH /tasks/{id} |
| `cancelled` | Выставляется вручную через PATCH /tasks/{id} |

## Для мобильной интеграции

- При получении свежесозданной задачи ожидай `"status": "new"` — это не ошибка
- Чтобы перевести задачу в другой статус, используй `PATCH /tasks/{id}` с телом `{ "status": "pending" }` (или нужный статус)
- При фильтрации по статусу (`GET /tasks?status=new`) используй `new`, не `pending`
